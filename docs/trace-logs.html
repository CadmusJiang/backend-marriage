🔍 全链路日志追踪器

通过Trace-ID追踪HTTP请求的完整执行链路，包括SQL查询和Redis操作

📖 使用说明

  • Trace-ID: 在请求头中添加 X-Trace-ID 或 Trace-ID 来标识请求
  • 自动生成: 如果没有提供Trace-ID，系统会自动生成一个
  • 全链路追踪: 系统会自动记录请求过程中的所有SQL查询和Redis操作
  • 响应头: 响应头中会包含 X-Trace-ID 字段
Trace-ID:
日志数量: 0
时间范围: -

输入Trace-ID或生成测试请求来查看全链路日志
